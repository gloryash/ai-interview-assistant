## 1. 文档概述

### 1.1 产品定位

AI 访谈助手是一个**通过语音访谈帮助用户将模糊想法转化为结构化文档**的工具。

核心价值：
- 降低需求表达门槛（语音比文字更自然）
- AI 主动引导（用户不需要知道文档怎么写）
- 自动生成文档（PRD、用户画像等）

### 1.2 用户研究方法

- 营地学员观察与访谈
- 产品经理社群调研
- 教育培训场景分析

## 2. 用户画像

### 2.1 画像一：小雨 - 创意型学生

#### 基本信息

| 属性 | 描述 |
|------|------|
| 姓名 | 小雨（化名） |
| 年龄 | 14 岁 |
| 身份 | 初二学生，营地学员 |
| 技术水平 | 零基础，不会编程 |
| 所在城市 | 北京 |

#### 特点与背景

小雨是一个充满创意的初中生，脑子里有很多天马行空的想法。她喜欢画画和讲故事，对 AI 工具很好奇，但完全不懂技术。

#### 痛点与需求

| 痛点 | 具体表现 | 期望解决方案 |
|------|----------|--------------|
| 不知道从何说起 | 想做一个帮同学背单词的 App，但不知道怎么描述 | AI 主动提问，引导表达 |
| 表达逻辑混乱 | 写需求时东一句西一句 | AI 帮助整理成结构化文档 |
| 不懂专业术语 | 不知道什么是 PRD、用户画像 | 只需要说话，AI 自动生成专业文档 |

#### 使用场景

```
场景：营地课程
小雨参加 AI 产品营，需要完成一个产品设计作业：
1. 打开 AI 访谈助手，选择"AI 产品访谈"
2. AI 友好地打招呼，问她想做什么产品
3. 小雨用语音描述自己的想法
4. AI 追问细节，帮她理清思路
5. 10 分钟后，自动生成 PRD 和用户画像
6. 小雨下载文档，用于后续开发
```

#### 期望的产品体验

- 像和朋友聊天一样自然
- 不需要学习任何专业知识
- 说完话就能得到专业文档

### 2.2 画像二：阿杰 - 技术型学生

#### 基本信息

| 属性 | 描述 |
|------|------|
| 姓名 | 阿杰（化名） |
| 年龄 | 16 岁 |
| 身份 | 高一学生，信息学竞赛选手 |
| 技术水平 | 熟练掌握 Python/C++ |
| 所在城市 | 深圳 |

#### 特点与背景

阿杰技术能力很强，能独立开发小项目。但他习惯直接写代码，不擅长需求分析，经常做到一半发现方向错了。

#### 痛点与需求

| 痛点 | 具体表现 | 期望解决方案 |
|------|----------|--------------|
| 边做边改 | 做项目时经常返工 | 开发前先用 AI 梳理需求 |
| 不擅长表达 | 不知道怎么向非技术人员解释想法 | AI 帮助生成易懂的文档 |
| 忽略用户视角 | 只考虑技术实现，忽略用户需求 | AI 引导思考用户场景 |

#### 使用场景

```
场景：项目启动
阿杰要开始一个新的 Side Project：
1. 开始写代码前，先用 AI 访谈助手
2. AI 引导他思考：解决什么问题？谁会用？
3. 通过对话，发现自己之前没想清楚的点
4. 生成 PRD，作为开发的指导文档
5. 避免了后续的大量返工
```

### 2.3 画像三：李老师 - 助教/教师

#### 基本信息

| 属性 | 描述 |
|------|------|
| 姓名 | 李老师（化名） |
| 年龄 | 28 岁 |
| 身份 | 营地助教 |
| 技术水平 | 中等，会基本操作 |
| 所在城市 | 上海 |

#### 特点与背景

李老师是一名营地助教，每天需要上课、辅导学生。课后需要进行教学复盘，但往往太忙没时间写详细的复盘报告。

#### 痛点与需求

| 痛点 | 具体表现 | 期望解决方案 |
|------|----------|--------------|
| 没时间写报告 | 课后很累，不想写长篇复盘 | 用语音快速复盘，AI 自动生成报告 |
| 复盘不系统 | 想到哪说到哪，容易遗漏 | AI 按框架引导，确保全面 |
| 难以坚持 | 复盘习惯难以养成 | 降低门槛，说几分钟话就行 |

#### 使用场景

```
场景：课后复盘
李老师刚上完一节课：
1. 打开 AI 访谈助手，选择"助教复盘"
2. AI 问：今天上的什么课？学生表现如何？
3. 李老师边走边用语音回答
4. AI 追问：遇到什么挑战？怎么解决的？
5. 5 分钟后，生成结构化的复盘报告
6. 报告包含亮点、改进点、下次行动计划
```

## 4. 产品设计启示

### 4.1 必须满足的需求

1. **语音优先** - 说话比打字更自然
2. **AI 主动引导** - 用户不需要知道该说什么
3. **自动生成文档** - 无需手动整理

### 4.2 场景优先级

| 优先级 | 场景 | 目标用户 |
|--------|------|----------|
| P0 | AI 产品访谈 | 学生、开发者 |
| P1 | 助教复盘 | 教师、助教 |
| P2 | 用户调研 | 产品经理 |
