## 1. 产品概述

### 1.1 产品定位

**一句话描述**：通过 AI 语音访谈，帮助用户将模糊的想法转化为结构化文档。

**核心价值**：
- 降低表达门槛（语音比文字更自然）
- AI 主动引导（用户不需要知道文档怎么写）
- 实时结构化（边聊边生成文档）
- 场景可扩展（一套代码，多种场景）

### 1.2 目标用户

| 用户类型 | 描述 | 使用场景 |
|---------|------|---------|
| 产品新手 | 有想法但不知道如何表达 | 梳理产品需求 |
| 开发者 | 习惯写代码，不擅长写文档 | 项目启动前梳理需求 |
| 助教/教师 | 需要进行教学复盘 | 课后反思总结 |
| 用户研究员 | 需要进行用户访谈 | 用户调研 |

## 3. 核心功能详解

### 3.1 语音对话

| 功能 | 说明 |
|-----|------|
| 语音识别 (ASR) | 阿里云 Paraformer 实时识别 |
| 语音合成 (TTS) | 阿里云 CosyVoice 语音合成 |
| 语音活动检测 (VAD) | 自动检测说话开始/结束 |
| 语音打断 | 用户说话时自动停止 AI 回复 |

### 3.2 场景选择

支持多种访谈场景，通过下拉菜单切换：
- AI 产品访谈（已实现）
- 助教复盘（可扩展）
- 用户调研（可扩展）

### 3.3 访谈引导

AI 按预设流程主动提问：
1. 开场寒暄
2. 核心信息收集
3. 追问细节
4. 总结确认

### 3.4 文档生成

访谈结束后自动生成：
- PRD 文档
- 用户画像
- 其他配置的文档

## 4. 技术架构

### 4.1 技术栈

| 层级 | 技术 |
|-----|------|
| 前端框架 | React 19 + TypeScript |
| 构建工具 | Vite 7 |
| 音频处理 | Web Audio API (AudioWorklet) |
| 实时通信 | WebSocket |
| 云服务 | 阿里云 DashScope |

### 4.2 核心设计

采用 **Prompt 驱动架构**：
- 一套代码，多种场景
- 通过配置文件定义场景行为
- 新增场景无需修改代码

详见：[05-Prompt驱动架构设计.md](./05-Prompt驱动架构设计.md)
